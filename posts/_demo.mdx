This file is a living reference of every component, syntax, and styling available when writing posts on howznguyen.dev. Use it to check how things render, or copy snippets into your own posts.

## Text formatting

Standard markdown works as expected. **Bold text**, *italic text*, and `inline code` all render with custom styling. You can also use ~~strikethrough~~ and combine [styled links](https://example.com) with regular text.

Here's a paragraph with multiple sentences to demonstrate line height and reading rhythm. The prose styling on the site gives body text a 1.85 line-height for comfortable reading. Paragraphs are separated by blank lines in the source, and render with generous spacing.

## Code blocks

Fenced code blocks with a language tag get Shiki syntax highlighting, line numbers, and a copy button:

```typescript
// A simple function
export function greet(name: string): string {
  if (!name) throw new Error("Name is required");
  return `Hello, ${name}!`;
}

// Usage
greet("Howz"); // "Hello, Howz!"
```

## Callouts

Callouts highlight important info. Six types are available:

<Callout type="info" title="Info">
  Use **info** for neutral asides and extra context.
</Callout>

<Callout type="tip">
  Use **tip** for helpful suggestions and best practices. Uses the default title.
</Callout>

<Callout type="warning" title="Heads up">
  Use **warning** when something might trip the reader up.
</Callout>

<Callout type="success">
  Use **success** to confirm the reader has done the right thing.
</Callout>

<Callout type="error" title="Don't do this">
  Use **error** to flag pitfalls or security issues.
</Callout>

<Callout type="note" emoji="🧩">
  Use **note** for neutral asides. Emoji can be overridden with the `emoji` prop.
</Callout>

## Putting it all together

A typical post mixes several of these. Here's a realistic paragraph:

When building the post editor we hit a tricky issue with hydration. The server was rendering [one state](https://react.dev/reference/react-dom/client/hydrateRoot) while the client rendered another, which caused a mismatch.

<Callout type="tip" title="Debugging hydration">
  Add `suppressHydrationWarning` sparingly. Most of the time the right fix is to make the server and client render identical output.
</Callout>

```typescript
export default function Layout({ children }: { children: ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>{children}</body>
    </html>
  );
}
```

See the [Next.js docs](https://nextjs.org/docs) for more.
