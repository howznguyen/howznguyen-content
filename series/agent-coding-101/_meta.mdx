Mở một coding agent lên thì khá dễ. Giao cho nó một câu ngắn, nhìn mấy dòng chữ chạy ào ào rồi thấy code xuất hiện cũng vui mắt lắm.

Khó hơn là để agent hiểu đúng project, nhận đúng task, sửa đúng chỗ và không kéo mình vào một buổi debug còn dài hơn lúc tự code. Chỗ này mới bắt đầu bớt vui.

**Agent Coding 101** là series mình viết cho những bạn mới bắt đầu dùng AI để code. Bạn dùng Claude Code, Codex, Cursor hay Copilot đều được. Mục tiêu không phải tìm ra một prompt thần kỳ, mà là xây một cách làm đủ rõ để mình biết agent đang làm gì và kiểm tra nó ra sao.

Series bắt đầu từ mấy khái niệm cơ bản, rồi đi dần vào cách giao việc, giúp agent hiểu project, biến một feature mơ hồ thành spec và chia thành các task nhỏ. Những bài sau nói về context, memory, nhiều agent cùng làm việc, cách review kết quả và lúc nào nên giành tay lái lại.

## Series này dành cho ai

Series này phù hợp nếu bạn đã biết code nhưng vẫn chưa thật sự thoải mái khi giao việc cho agent. Có thể bạn từng viết một prompt rất dài mà kết quả vẫn sai. Hoặc agent làm đúng ở vài file đầu, rồi bắt đầu quên giới hạn, sửa lan sang chỗ khác và để lại một diff nhìn thôi đã muốn đi pha cà phê.

Bạn không cần biết quá sâu về LLM hay phải dùng đúng một tool nào đó. Chỉ cần có một project thật và sẵn sàng bắt đầu bằng những task nhỏ.

## Bạn sẽ học được gì

Mình sẽ đi từ những tình huống thường gặp, giải thích vì sao chúng xảy ra và đưa ra cách xử lý có thể thử ngay trên project thật.

Sau series này, bạn sẽ biết cách giao một task có phạm vi rõ, cung cấp đủ context, viết sổ tay project vừa đủ và review kết quả thay vì chỉ nhìn xem code có chạy hay không. Khi feature lớn hơn, bạn cũng sẽ biết cách đi từ ý tưởng tới spec, chia thành các task nhỏ và nhận ra lúc nào một session hoặc một agent không còn đủ.

Mục tiêu cuối cùng không phải là để agent viết được nhiều code hơn. Mà là để bạn kiểm soát được agent đang làm gì, vì sao nó làm như vậy và cách kiểm tra kết quả trước khi merge.

## Cách đọc series

Bạn nên đọc theo thứ tự nếu mới bắt đầu. Nếu đã dùng coding agent một thời gian, bạn có thể nhảy thẳng tới bài đang đúng với vấn đề mình gặp.

Mỗi bài tập trung vào một vấn đề và có thể đọc riêng. Nếu chưa biết bắt đầu ở đâu, cứ đi theo thứ tự các chương bên dưới. Mình đã xếp đường rồi, các bạn không cần tự vẽ thêm roadmap cho mệt.
