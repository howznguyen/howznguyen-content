# vi

Database transaction gom nhiều thao tác thành một đơn vị “làm hết hoặc không làm gì cả”. Nếu một bước thất bại, các thay đổi trong transaction có thể được quay lại để database không mắc kẹt ở trạng thái nửa vời.

Tìm hiểu thêm: [PostgreSQL - Transactions](https://www.postgresql.org/docs/current/tutorial-transactions.html).

# en

A database transaction groups several operations into a single all-or-nothing unit. If a step fails, the transaction can be rolled back so the database is not left in a partially updated state.

Learn more: [PostgreSQL - Transactions](https://www.postgresql.org/docs/current/tutorial-transactions.html).
