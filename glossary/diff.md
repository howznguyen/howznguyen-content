# vi

Diff là phần thể hiện sự khác nhau giữa hai trạng thái của code, thường cho biết dòng nào được thêm, xoá hoặc sửa. Review diff là đọc chính những thay đổi đó thay vì đọc lại cả repository từ đầu.

Tìm hiểu thêm: [Git - git-diff Documentation](https://git-scm.com/docs/git-diff.html).

# en

A diff represents the differences between two states of code, usually showing which lines were added, removed, or changed. Reviewing a diff means inspecting those changes instead of rereading the entire repository.

Learn more: [Git - git-diff Documentation](https://git-scm.com/docs/git-diff.html).
